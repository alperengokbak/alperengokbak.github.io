# Trip Registration Form Project

This is a simple web application for a trip registration form. Users can fill out the form, providing details such as their name, email, phone number, address, departure and destination points, preferred travel vehicle, and travel dates. The application performs various client-side validations to ensure that the user input is accurate and complete.

## Project Structure

The project consists of the following files:

### `index.html`

This is the main HTML file containing the structure of the trip registration form. It includes form elements for user input, JavaScript for form validation, and styles to enhance the user interface.

### `script.js`

This JavaScript file provides client-side validation for the form. It includes functions to toggle the navigation menu, display error messages, and validate user input for email, phone number, departure, destination points, checkbox agreement, and travel dates.

### `styles.css`

The CSS file styles the HTML elements, improving the visual appeal of the trip registration form. It defines the layout, colors, and appearance of various form elements.

## How to Run

1. Open the `index.html` file in a web browser.

2. Fill out the form with the required information.

3. Submit the form to register for the trip.

## Features

- **Navigation Menu:** The top navigation menu provides links to different sections of the website.
- **Responsive Design:** The application is designed to be responsive, ensuring a consistent user experience across various devices.
- **Form Validation:** Client-side validation is implemented to ensure that user input is accurate and complete.
- **Error Handling:** Error messages are displayed to users if they enter invalid or incomplete information.
- **Date Selection:** Users can choose their preferred travel dates using the date input fields.
- **Dropdown Menus:** Dropdown menus are provided for selecting departure points, destination points, and preferred travel vehicles.

## Data Fetching

The application fetches data from mock APIs to dynamically populate dropdown menus for departure points, destination points, and vehicle types.
